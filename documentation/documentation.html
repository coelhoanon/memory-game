Game logic

ignore if you know how a common memory game works

the game itself works as follows ... the player enters the game and all cards are created face up, and after a while they turn around and the game begins, when the player clicks on a card it turns, after clicking on two cards the algorithm checks if they are the same, if so, they are turned over until the end of the game, otherwise, the two cards turn over again. If the player hits all the cards before time runs out, he wins, otherwise he loses.

Gold power up

available in the archives "match cards.js", "cash system.js" and "gold_power_up.js"

The player always receives some amount of gold when he hits a pair of cards. The value of the gain is random between 1 and 10, and this value is passed to the cash system.js file


In the cash system.js file it will receive the cash value and update it in the local storage and html, but, if the gold power up is activated it will multiply by 2 before doing the same process.


The gold_power_up.js file basically does two things, activates the double gold effect and updates the amount of this power up in the local storage and html. SetInterval () was used to count the time when the power up is activated, and this is what updates the state of true or false of the power up.

Spin cards power up

available in the archive "spin_power_ups.js"

It saves which cards have not yet been tapped and then shows them for time the power up lasts, and it is important that all cards lose the click event to that there is no bug. When the power up time ends, the click event returns and the cards are turned over to the initial state.

XP power up

available in the archive "xp_power_ups.js" and "level system.js"

When activated, it triggers a function that starts a timer by setInterval (), and as long as the power up time lasts, the effect will be true, which in the level system.js file uses to multiply the xp's progress by the power up, and when the power up ends it turns false and everything goes back to normal.

Creating the cards

available in the archive "cards creator.js"

There are only two paths in creating the cards, creating them with the standard images or with market. The code checks in the storage location if there is a market deck being used or not, to define whether the cards will be standard or not.


Before letters were actually created in html, the array that is used to store the letters is drawn, making the cards now all shuffled every time html elements (letters) are created.


Right after the cards are shuffled and created, they are given an id in the value attribute of the cards to identify the pairs of cards. It works like this ... two FOR loops are made to compare the image of each card, and if the card images are the same, they are given an id.


The cards are created with the face open, so that the player can memorize the cards before starting to play (that's why it's called the memory game, not the guessing game), when all the cards have already been created, they are turned over for a certain time (2200) and only then do they turn around.

Match of cards

available in the archive "match cards.js"

First, the number of cards turned is checked, so that it is not possible for the user turn more than two cards at a time. When there are no cards dealt, the spin function is activated, and when there is already a card turned over, verification is made that they are the same. If there is a match between the cards the player wins gold and XP.

Game pause system

available in the archive "timer.js"

A setInterval () was used that will be executed until the end of the game. The "pause_play" variable is responsible for controlling the game clock; whenever it has the value "PLAY" the counter will be reduced until it reaches zero.

End of the game

available in the archive "end memory game.js"

The game ends only on two occasions, when time runs out or when the user hits all pairs of cards, and when that happens, the click event is removed from the cards, preventing the player from continuing to play and a screen with the result of the game (loser or winner).

Loser screen

available in the archive "end memory game.js"

The loser screen is displayed only when playing time ends before the player hits all the pairs of cards.

Winner screen

available in the archive "end memory game.js"

The winner screen is shown only when the player hits all pairs of cards before the game time ends. Every time a pair of cards is hit, the clear_game () function is called to verify that the number of cards matched corresponds to the total number of cards, and if true, the end-of-game code for the winner is activated.
